# Source: CMU Snow Plow Routing

## Summary

Carnegie Mellon University research on the Snow Plow Routing Problem (SPRP) using constraint programming. The SPRP involves finding vehicle routes to service a street network in a pre-defined area while respecting vehicle constraints and traffic restrictions. The underlying problem is the [[Capacitated Arc Routing|Capacitated Arc Routing Problem (CARP)]].

## Key Results

- **3% to 156% shorter routes** than routes generated by a city's commercial routing software
- Test pilot conducted with actual vehicles in a real municipal setting
- Constraint programming approach outperformed commercial tools across all test instances

## Key Distinction: Arc vs Node Routing

This paper highlights that snow plow (and by extension, grader) routing is fundamentally an **arc routing** problem: vehicles must traverse road segments (arcs), not visit point locations (nodes). Standard VRP solvers like [[PyVRP]] solve node-based problems. Applying VRP to arc-based problems requires transforming arc demands into node demands (e.g., representing each road segment as a node at its midpoint), which is a known approximation.

## Relevance to Road Intelligence Platform

The RIP's [[Route Planner]] uses [[PyVRP]] (a node-based VRP solver). This paper suggests that a CARP formulation may be more natural for grader routing. The arc-to-node transformation the RIP implicitly uses is standard practice, but awareness of the distinction matters for solution quality and for interpreting solver output.
